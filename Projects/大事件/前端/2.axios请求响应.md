## Axios插件
**唯一作用就是发 HTTP 请求、收响应**。前端通过它向后端 API 要数据（GET）或提交数据（POST/PUT/DELETE），然后把返回的数据更新到页面上，整个过程**不刷新页面**，这就是 AJAX 思想的具体落地。是前端页面和后端服务器之间数据的传输者。值得注意的是使用Axios一定会遇见的[[零散碎片待整理#跨域问题|跨域问题介绍]]
```shell
npm install axios                 // 下载Axios包到项目的node_modules目录：
import axios from 'axios'         // 引入到script中
```

默认参数
- axios.post(url, data, config)
- axios.get(url, config)

简单示例
```javascript
<script setup lang="ts">
	axios.get("url", {
		// config设置,规定如何传参，请求头配置等等
		param: {
			// GET请求参数必须放在params里，axios会自动拼接到URL末尾
			// 例如 /users?page=1&pageSize=10&keyword=张三
			....
		}
	}).then(result=>{
		// 请求成功处理
		xxx = result.data
	}).catch(err=>{
		//错误处理
		console.log(err)
	})
</script>
```



utils/request.js    # 拦截器

这个文件是整个前端项目的**网络通信核心**，它基于 Axios 封装了一个请求实例，统一处理了请求发送和响应接收的逻辑。

## 整体功能
创建一个配置好的 Axios 实例，并通过**请求拦截器**和**响应拦截器**，为项目中的所有 HTTP 请求提供两个核心能力：自动携带身份令牌和处理统一的响应错误。

## 实例创建
```
import axios from 'axios'; 
const baseURL = '/api'; 
const instance = axios.create({ baseURL }) 
```
- **`axios.create`**：创建了一个名为 `instance` 的、拥有自定义配置的 Axios 副本。
- **`baseURL: '/api'`**：提取公共请求前缀。此后，所有通过 `instance` 发送的请求地址，都会自动在前面加上 `/api`。例如调用 `instance.get('/user')` 实际请求的是 `/api/user`。[[0.Vite项目#Vite配置文件|api实际地址]] **这样可以灵活配置后端服务器地址**

## 配置拦截器
- 请求拦截器：instance.interceptors.request.use
- 响应拦截器：instance.interceptors.response.use

## 请求拦截器说明

比如注册服务：
```js
export const userRegisterService = (registerData)=>{
	// 1. 创建一个 URLSearchParams 实例
	const params = new URLSearchParams()
	// 2. 遍历 registerData 对象的所有属性
	for(let key in registerData){
	    // 3. 把每个键值对添加到 params 中
	    params.append(key, registerData[key]); 
	}
	// 4. 发送 POST 请求，params 会自动被序列化为字符串
	return request.post('/user/register', params);
}
```

**拦截器不是改变参数位置，而是在请求发出前，自动往 config 里添加东西，他不是拦截之后不发送，而是拦截之后往里面加东西**

```js
// 请求拦截器
instance.interceptors.request.use(
    (config)=>{
        //请求前的回调
        //查看token状态
        const tokenStore = useTokenStore();
        
        if(tokenStore.token){ //判断有没有token
            config.headers.Authorization = tokenStore.token
        }
        return config;
    },
    (err)=>{ 
        //请求错误的回调，很少触发）
        Promise.reject(err)
    }
)
```

所以无论你怎么调用 `request.post(url, data)`，都相当于`axios.post(url,data,config)`;
拦截器都会自动在 `config.headers` 里加上 token，你不需要每次都手动传 config。

```js
request.post('/user/register', params)
// ↓ 实际执行
axios.post('/user/register', params)  // config 为空对象
// ↓ 拦截器处理后
axios.post('/user/register', params, { headers: { Authorization: tokenStore.token } })
```

然后后端拦截器再对请求拦截，选择性放行


## 响应拦截器
```js
// 拦截器写法
instance.interceptors.response.use(
    result => {
        // 请求成功的回调
        return result.data
    },
    err => {
        // 请求错误的回调 ← 这个就是
        Promise.reject(err)
    }
)

// 相当于在每个请求后面自动加了 .catch
request.post('/user/register', params)
    .then(res => {
        // 成功处理
    })
    .catch(err => {  // ← 就是这个
        // 错误处理
    })
```
