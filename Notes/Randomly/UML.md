# 阶段
软件开发通常分为6个核心阶段，每个阶段常用的UML图如下，一目了然：

**1. 需求分析（搞清楚要做什么）**
- 用例图：画清楚“谁”能“用系统干什么”，框定系统边界和功能。
- 活动图：梳理业务流程（如审批流），适合描述复杂操作步骤。

**2. 系统设计（规划系统架构）**
- 类图：核心！定义系统中的类、属性、方法及相互关系。
- 包图：将大量类分包管理，展示系统的高层模块依赖。
- 部署图：把软件组件部署到硬件节点上，展示物理拓扑。
    
**3. 详细设计（打磨内部交互细节）**
- 顺序图：最常用。按时间线展示对象间的消息传递与调用顺序。
- 通信图（协作图）：也展示对象交互，但更突出“谁跟谁有连接”，是顺序图的视角补充。
- 状态机图：分析对象在生命周期内的各种状态及转换条件（防漏单神器）。
    
**4. 实现/编码（把图转成代码）**
- 组件图：展示代码的物理模块（如jar包、dll、源文件）及接口依赖，指导构建。

**5. 测试（验证对不对）**
- 用例图：直接拿需求阶段的图导出测试用例。
- 活动图：为复杂逻辑路径设计测试流。
- 顺序图：验证系统运行时对象间的调用是否符合预期。
    

**6. 部署与维护**
- 部署图：编写实际的物理部署方案。
- 组件图：管理版本与依赖，辅助运维。
    
**快速记忆诀窍：**
- 画**功能**给客户看 → **用例图**
- 画**流程**给领导看 → **活动图**
- 画**结构**给程序员看 → **类图**
- 画**交互**排错看 → **顺序图**
- 画**状态**防漏单 → **状态机图**
- 画**部署**给运维看 → **部署图**

# 图

## 包图 Package
依赖：引入(import)


## 组件图 Component
系统设计的一个模块化部分

接口：原子化的两两组件的关系(出口和入口)

端口：封装后的多个接口关系所组成的组件 对外展示的接口(分为入口和出口)


## 部署图 Deployment

组件的上层封装
