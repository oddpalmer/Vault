# 进程调度
## Time

| 时间                 | 定义                        |
| ------------------ | ------------------------- |
| 到达时间               | 进程进入就绪队列的时间               |
| 执行时间 / 服务时间        | 进程需要的 CPU 执行时间            |
| 完成时间               | 进程执行完毕的时间                 |
| 响应时间               | 从提交到首次获得 CPU 的时间，对交互式系统重要 |
| 周转时间 = 完成时间 - 到达时间 | 衡量作业从提交到完成的总耗时            |
| 等待时间 = 周转时间 - 执行时间 | 表示进程在就绪队列中等待 CPU 的总时间     |
|                    |                           |

**先来先服务**
First-Come, First-Served
按作业到达顺序调度，简单
缺点：平均等待时间可能更长

**最短作业优先**
Shortest Job First
在空闲时刻，优先调度预计运行时间最短的作业，可降低平均等待时间，但需预知执行时间
缺点：使运行时间长的JOB处于饥饿状态

**时间片轮转**
Round Robin
每个进程分配固定时间片，超时则轮转，适合分时系统，保证公平性。
缺点：时间片过长 会退化为FCFS； 时间片过短则会增加开销

**优先级调度**
Priority Scheduling
按优先级调度，高优先执行，可抢占或非抢占，可能造成低优先级饥饿。
缺点：若无老化机制，低优先级进程可能长期得不到执行。

**多级反馈队列**
Multilevel Feedback Queue
多队列+动态优先级+时间片调整，兼顾响应与吞吐，适用于通用系统。
缺点：实现复杂，参数配置敏感，调优困难。
