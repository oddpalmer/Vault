# 山火识别监控系统 - API 接口文档

**版本**: v1.0
**更新日期**: 2026-04-27
**描述**: 本接口文档定义了山火识别监控系统后端服务与前端应用之间的交互规范。

## 通用说明

- **基础路径**: `/api/v1`
- **协议**: HTTPS
- **数据格式**: JSON
- **字符编码**: UTF-8
- **认证方式**: JWT (Bearer Token)
    - 请求头需包含: `Authorization: Bearer <token>`

## 通用响应结构

所有接口响应均遵循以下统一结构：

- **200 OK**: 请求成功
- **400 Bad Request**: 客户端参数错误
- **401 Unauthorized**: 认证失败或 Token 过期
- **403 Forbidden**: 权限不足
- **404 Not Found**: 资源不存在
- **500 Internal Server Error**: 服务器内部错误

```json
{
  "code": 200,
  "success": true,
  "message": "操作成功",
  "data": {}
}
```


### 全局错误码

|错误码|HTTP状态|描述|
|---|---|---|
|1000|400|参数校验失败|
|1001|401|用户未登录或登录已过期|
|1002|403|无权访问该资源|
|1003|404|请求的资源不存在|
|9999|500|系统内部异常，请联系管理员|

## 1. 用户认证与管理接口

### 1.1 用户登录

- **接口描述**: 用户登录系统，获取 JWT Token
- **请求方式**: POST
- **路径**: `/auth/login`
- **认证**: 无需 Token

#### 请求参数

|参数名|类型|必填|描述|
|---|---|---|---|
|username|string|是|用户名|
|password|string|是|密码 (MD5加密传输)|

#### 响应数据

|字段名|类型|描述|
|---|---|---|
|token|string|JWT 认证令牌|
|expireIn|int|有效期(秒)|

### 1.2 获取当前用户信息

- **接口描述**: 获取当前登录用户的基本信息及权限
- **请求方式**: GET
- **路径**: `/auth/profile`
- **认证**: 需要 Token

#### 响应数据

|字段名|类型|描述|
|---|---|---|
|userId|int|用户ID|
|username|string|用户名|
|role|string|角色 (ADMIN, OPERATOR, VIEWER)|
|email|string|邮箱|

### 1.3 创建新用户

- **接口描述**: 管理员创建新用户
- **请求方式**: POST
- **路径**: `/users`
- **认证**: 需要 Token (ADMIN权限)

#### 请求体

```json
{
  "username": "new_user",
  "password": "encrypted_password",
  "email": "user@example.com",
  "role": "OPERATOR"
}
```

#### 响应数据

|字段名|类型|描述|
|---|---|---|
|userId|int|新创建用户的ID|
|message|string|操作结果消息|

### 1.4 更新用户角色权限

- **接口描述**: 更新指定用户的角色权限
- **请求方式**: PUT
- **路径**: `/users/{id}/role`
- **认证**: 需要 Token (ADMIN权限)

#### 路径参数

|参数名|类型|必填|描述|
|---|---|---|---|
|id|int|是|用户ID|

#### 请求体

```json
{
  "role": "ADMIN"
}
```

### 1.5 修改用户密码

- **接口描述**: 修改指定用户密码
- **请求方式**: PUT
- **路径**: `/users/{id}/password`
- **认证**: 需要 Token (ADMIN权限或本人)

#### 路径参数

|参数名|类型|必填|描述|
|---|---|---|---|
|id|int|是|用户ID|

#### 请求体

```json
{
  "oldPassword": "current_password", // 仅本人修改时需要
  "newPassword": "new_encrypted_password"
}
```

## 2. 预警管理接口

### 2.1 获取实时预警列表

- **接口描述**: 分页查询预警信息，支持按等级和状态过滤
- **请求方式**: GET
- **路径**: `/alerts`
- **认证**: 需要 Token

#### 查询参数

|参数名|类型|必填|描述|默认值|
|---|---|---|---|---|
|page|int|否|页码|1|
|size|int|否|每页数量|10|
|level|string|否|预警等级 (BLUE, YELLOW, ORANGE, RED)|无|
|status|string|否|状态 (ACTIVE, CANCELLED, HANDLED)|ACTIVE|

#### 响应数据 (data为数组)

|字段名|类型|描述|
|---|---|---|
|id|int|预警ID|
|location|string|地理位置描述|
|latitude|number|纬度|
|longitude|number|经度|
|level|string|预警等级|
|status|string|状态|
|confidence|number|AI识别置信度|
|imageUrl|string|现场截图URL|
|createTime|string|创建时间 (ISO8601)|

- **接口描述**: 操作员确认预警属实并开始处理
- **请求方式**: PUT
- **路径**: `/alerts/{id}/handle`
- **认证**: 需要 Token

#### 路径参数

|参数名|类型|必填|描述|
|---|---|---|---|
|id|int|是|预警ID|

#### 请求体

```json
{

}
```

### 2.3 发送预警通知

- **接口描述**: 发送预警通知给相关人员
- **请求方式**: POST
- **路径**: `/alerts/notifications`
- **认证**: 需要 Token

#### 请求体

```json

  "notificationType": "SMS",
  "recipients": ["13800138000", "admin@forest.com"],
  "content": "山火预警：地点XX，等级红色，请立即处理"
```

### 2.4 获取预警历史记录

- **接口描述**: 获取预警历史记录
- **请求方式**: GET
- **路径**: `/alerts/history`

#### 查询参数

|level|string|否|预警等级|无|
