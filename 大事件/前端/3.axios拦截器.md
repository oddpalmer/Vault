**`utils/request.js`** # 网络请求核心模块

这个文件是整个前端项目的**网络通信核心**，它基于 Axios 封装了一个请求实例，统一处理了请求发送和响应接收的逻辑。

------

### 整体功能
创建一个配置好的 Axios 实例，并通过**请求拦截器**和**响应拦截器**，为项目中的所有 HTTP 请求提供两个核心能力：自动携带身份令牌和处理统一的响应错误。

### 1. 实例创建
```
import axios from 'axios'; 
const baseURL = '/api'; 
const instance = axios.create({ baseURL }) 
```
- **`axios.create`**：创建了一个名为 `instance` 的、拥有自定义配置的 Axios 副本。
- **`baseURL: '/api'`**：提取公共请求前缀。此后，所有通过 `instance` 发送的请求地址，都会自动在前面加上 `/api`。例如调用 `instance.get('/user')` 实际请求的是 `/api/user`。[[Vue3#Vite配置文件]]

### 2. 请求拦截器 (request interceptor)
```
instance.interceptors.request.use( (config) => { ... } ) 
```
这是请求发出前必须经过的关口，主要做一件事：**自动附加 Token**。

- **`(config) => { ... }`**：请求前的回调函数，基本都会用这个。
- **`useTokenStore()`**：从 Pinia Store 引入并获取 token 的响应式状态。
- **`config.headers.Authorization`**：核心逻辑。如果 Store 中存在 token，就将其挂载到本次请求的 `Authorization` 请求头上，后端以此验证用户身份。这避免了在每个接口调用处手动添加 Token。

```javascript
// 请求前的回调，主要看这里
(config)=>{
    const tokenStore = useTokenStore();
    if(tokenStore.token){
        config.headers.Authorization = tokenStore.token  // 关键操作：加 Token
    }
    return config;
}
```

### 3. 响应拦截器 (response interceptor) - 业务层
```
instance.interceptors.response.use( result => { ... } ) 
```
这是请求成功后，数据进入 `.then` 前的关口，主要处理**业务状态码**。

- **`result => { ... }`**：成功的回调函数。
- **`result.data.code === 0`**：这是与服务端约定的**业务成功状态码**。如果 code 为 0，函数直接 **`return result.data`**，即只把后端返回的有效数据体传递给真正的业务代码，简化了后续处理。
- **操作失败处理**：如果业务状态码不是 0，则直接视作操作失败：
  - **`ElMessage.error(...)`**：使用 Element Plus 的消息组件，弹出后端返回的失败信息 (`result.data.message`)。
  - **`return Promise.reject(result.data)`**：将本次请求 Promise 链的状态强行转变为失败，这样业务代码里的 `.catch` 就可以捕获到。

```javascript
result => {
    // 判断业务状态码, 成功逻辑
    if(result.data.code === 0){
        return result.data;
    }
    // 失败逻辑
    ElMessage.error(result.data.message)
    return Promise.reject(result.data)  // 关键操作：转为失败状态
}
```

### 4. 响应拦截器 (response interceptor) - HTTP 层
```
instance.interceptors.response.use( ..., err => { ... } ) 
```
这是请求失败，进入 `.catch` 前的关口，主要处理**HTTP 状态码异常**，比如网络问题或权限问题。

- **`err => { ... }`**：失败的拦截回调。
- **401 状态码特殊处理**：`if(err.response.status===401)` 专门捕获“未授权”错误。这通常意味着 Token 失效或未登录，处理方式是：
  - 弹出“请先登录”的提示。
  - 通过 **`router.push('/login')`** 将页面强制重定向到登录页。
- **其他错误统一处理**：除了 401 以外的错误，统一用 “服务异常” 来兜底提示。
- **`return Promise.reject(err)`**：最终仍将错误向下传递。

```javascript
err => {
    // 判断响应状态码, 401 特殊处理
    if(err.response.status === 401){
        ElMessage.error('请先登录')
        router.push('/login')  // 关键操作：跳转登录页
    } else {
        ElMessage.error('服务异常')
    }
    return Promise.reject(err); // 继续传递错误
}
```

这个文件最终通过 `export default instance;` 导出配置好的实例，供项目中所有 API 接口函数调用。
