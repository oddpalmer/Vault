## **1. 导入依赖**
```javascript
import { Edit, Delete } from '@element-plus/icons-vue'
import { ref } from 'vue'
```
- `Edit, Delete`：导入 Element Plus 的编辑和删除图标组件，用于表格操作按钮
- `ref`：Vue 3 的响应式数据 API，用于创建可响应的数据变量

---

## **2. 数据模型定义**
```javascript
const categorys = ref([])        // 存储文章分类列表
const categoryId = ref('')      // 用户选中的分类ID（搜索条件）
const state = ref('')          // 用户选中的发布状态（搜索条件）
const articles = ref([])       // 文章列表数据
```

这些是页面的核心数据，使用 `ref` 包装使其具有响应式特性，数据变化时页面自动更新。

## **3. 分页事件处理**
```javascript
const onSizeChange = (size) => {
    pageSize.value = size    // 更新每页条数
    articleList()           // 重新加载文章列表
}

const onCurrentChange = (num) => {
    pageNum.value = num     // 更新当前页码
    articleList()          // 重新加载文章列表

```
- 当用户改变每页显示条数或切换页码时触发
- 更新对应数据后重新请求后端数据

## **4. 获取文章分类**
```javascript
import { articleCategoryListService, articleListService, articleAddService } from '@/api/article.js'

const articleCategoryList = async () => {
    let result = await articleCategoryListService();  // 调用API获取分类
    categorys.value = result.data;                   // 存储分类数据
}
```
- 从后端 API 导入三个接口方法
- `articleCategoryList` 函数：调用分类接口，获取所有文章分类，用于下拉选择框

---

## **5. 获取文章列表**
```javascript
const articleList = async () => {
    let params = {
        pageNum: pageNum.value,
        pageSize: pageSize.value,
        categoryId: categoryId.value ? categoryId.value : null,  // 如果为空则传null
        state: state.value ? state.value : null
    }
    let result = await articleListService(params);  // 带参数请求
    
    total.value = result.data.total;      // 更新总条数（用于分页）
    articles.value = result.data.items;   // 更新文章列表数据
```
这个函数：
- 收集当前分页和筛选条件作为参数
- 请求后端接口获取文章列表
- 将返回数据赋值给对应变量，更新页面显示

---

## **6. 数据关联处理**
```javascript
    // 循环遍历每篇文章
    for (let i = 0; i < articles.value.length; i++) {
        let article = articles.value[i];
        // 再遍历所有分类
        for (let j = 0; j < categorys.value.length; j++) {
            // 如果文章的分类ID匹配，添加分类名称属性
            if (article.categoryId == categorys.value[j].id) {
                article.categoryName = categorys.value[j].categoryName;
            }
        }
    }
}
```
这段代码的作用：
- 后端返回的文章只有 `categoryId`（数字），没有分类名称
- 通过双层循环，将数字 ID 转换成可读的分类名称
- 例如：`categoryId: 1` → `categoryName: "技术文章"`

---

## **7. 页面初始化**
```javascript
articleCategoryList()  // 先加载分类列表
articleList();        // 再加载文章列表
```
- 页面加载时立即执行这两个函数
- 必须先加载分类，因为文章列表数据需要用到分类名称

## **11. 添加文章功能**

## **17. 图片上传组件**
`el-upload` 组件在设置了 `action` 和 `auto-upload="true"` 后，会**自动使用自己的 XMLHttpRequest 发送上传请求**，而不是走你项目里封装的 axios 实例。
```html
<el-upload 
    :auto-upload="true"                          // 选择文件后自动上传
    :show-file-list="false"                      // 不显示文件列表
    action="/api/upload"                         // 上传接口地址
    name="file"                                  // 文件字段名
    :headers="{'Authorization':tokenStore.token}" // 请求头携带token
    :on-success="uploadSuccess"                  // 上传成功回调
>
    <img v-if="articleModel.coverImg" :src="articleModel.coverImg" class="avatar" />
    <el-icon v-else class="avatar-uploader-icon"><Plus /></el-icon>
</el-upload>
```
- 如果已上传封面则显示图片，否则显示加号图标
- 自动上传到后端接口
