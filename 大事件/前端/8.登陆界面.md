## **1. 导入依赖**
```javascript
import { User, Lock } from '@element-plus/icons-vue'
import { ElMessage } from 'element-plus'
import { ref } from 'vue'
```
- `User, Lock`：从 Element Plus 图标库中导入“用户”和“锁”的图标组件，分别用于用户名和密码输入框的前缀图标，提升视觉效果。
- `ElMessage`：导入 Element Plus 的消息提示组件，用于在操作成功后（如注册、登录）弹出友好的提示框。
- `ref`：Vue 3 的核心响应式 API，用于创建基本类型的响应式数据。

## **2. 控制表单显示与数据模型定义**
```javascript
//控制注册与登录表单的显示， 默认显示注册
const isRegister = ref(false)
```
- `isRegister`：一个布尔类型的响应式数据，用于切换登录和注册表单的显示。`false` 表示默认显示登录表单。

```javascript
//定义数据模型
const registerData = ref({
    username: '',
    password: '',
    rePassword: ''
})

const loginData = ref({
    username: '',
    password: ''
})
```
- `registerData`：注册表单的数据模型，包含用户名、密码和确认密码三个字段，与注册表单的输入框进行双向绑定。
- `loginData`：登录表单的数据模型，包含用户名和密码两个字段，与登录表单的输入框进行**双向绑定**。将登录和注册的数据模型分开定义，使逻辑更清晰。

## **3. 自定义表单校验规则**
```javascript
// 前端校验数据登陆数据的函数
const checkRePassword = (rule, value, callback) => {
    if (value === '') {
        callback(new Error('请再次确认密码'))
    } else if (value !== registerData.value.password) {
        callback(new Error('请确保两次输入的密码一样'))
    } else {
        callback()
    }
}
```
- 这是一个自定义的校验函数，专门用于“确认密码”字段。
- 它接收三个参数：`rule`（校验规则）、`value`（当前输入值）、`callback`（回调函数）。
- **逻辑**：
    - 如果未输入，调用 `callback` 并传入一个错误对象，提示“请再次确认密码”。
    - 如果输入的值与 `registerData` 中的 `password` 不一致，提示“两次输入的密码不一样”。
    - 如果校验通过，直接调用 `callback()` 不传参数，表示通过。

```javascript
// Element表单规则，定义表单校验规则, 注册和更改密码都可以用吧
const rules = {
    username: [
        { required: true, message: '请输入用户名', trigger: 'blur' },
        { min: 5, max: 16, message: '长度为5~16位非空字符', trigger: 'blur' }
    ],
    password: [
        { required: true, message: '请输入密码', trigger: 'blur' },
        { min: 5, max: 16, message: '长度为5~16位非空字符', trigger: 'blur' }
    ],
    rePassword: [
        { validator: checkRePassword, trigger: 'blur' }
    ]
}
```
- 这是 Element Plus 表单组件所需的校验规则对象。
- `username` 和 `password` 都包含两个规则：`required`（必填）和 `min/max`（长度限制）。
- `rePassword` 没有使用内置规则，而是使用了上面定义的 `checkRePassword` 自定义校验器。
- `trigger: 'blur'` 表示在输入框失去焦点时触发校验。

## **4. 调用后台接口完成注册**

```javascript
//调用后台接口,完成注册
import { userRegisterService, userLoginService} from '@/api/user.js'
const register = async () => {
    //registerData是一个响应式对象,如果要获取值,需要.value
    let result = await userRegisterService(registerData.value);
    ElMessage.success(result.msg ? result.msg : '注册成功')
}
```
- 从 `@/api/user.js` 文件中导入注册和登录的接口服务函数。
- `register` 函数：
    - 使用 `async/await` 异步调用 `userRegisterService` 接口，并将 `registerData.value`（注册表单数据）作为参数传入。
    - 请求成功后，使用 `ElMessage.success` 弹出成功提示，优先显示后端返回的消息 `result.msg`，如果后端没提供，则显示默认的“注册成功”。

## **5. 登录功能实现**
```javascript
//绑定数据,复用注册表单的数据模型
//表单数据校验
//登录函数
import {useTokenStore} from '@/stores/token.js'
import {useRouter} from 'vue-router'
const router = useRouter()
const tokenStore = useTokenStore();
const login =async ()=>{
    //调用接口,完成登录
   let result =  await userLoginService(loginData.value);
   ElMessage.success(result.msg ? result.msg : '登录成功')
   //把得到的token存储到pinia中
   tokenStore.setToken(result.data)
   //跳转到首页 路由完成跳转
   router.push('/')
}
```
- **导入模块**：
    - `useTokenStore`：导入 Pinia 的 token 状态管理仓库。
    - `useRouter`：导入 Vue Router 的路由实例，用于编程式导航。
- **获取实例**：创建 `router` 和 `tokenStore` 实例。
- `login` 函数：
    - 调用 `userLoginService` 接口，传入 `loginData.value`。
    - 登录成功后，弹出成功提示。
    - **Token 持久化**：调用 `tokenStore.setToken(result.data)` 将后端返回的 token 存储到 Pinia 仓库中（通常也会同步存储到 `localStorage`，以便页面刷新后依然能保持登录状态）。
    - **路由跳转**：使用 `router.push('/')` 跳转到系统首页。

## **6. 数据清空函数**
```javascript
//定义函数,清空数据模型的数据
const clearRegisterData = ()=>{
    registerData.value={
        username:'',
        password:'',
        rePassword:''
    }
}

const clearLoginData = () => {
    loginData.value = {
        username: '',
        password: ''
    }
}
```
- 这两个函数用于重置表单数据模型。
- 当用户点击“返回登录”或“去注册”链接时，清空相应的表单数据，防止之前输入的内容残留，提升用户体验。

## **8. 注册表单模板**
```html
<!-- 注册表单 -->
<el-form ref="form" size="large" autocomplete="off" v-if="isRegister" :model="registerData" :rules="rules">
    <el-form-item>
        <h1>注册</h1>
    </el-form-item>
    <el-form-item prop="username">
        <el-input :prefix-icon="User" placeholder="请输入用户名" v-model="registerData.username"></el-input>
    </el-form-item>
    <el-form-item prop="password">
        <el-input :prefix-icon="Lock" type="password" placeholder="请输入密码" v-model="registerData.password"></el-input>
    </el-form-item>
    <el-form-item prop="rePassword">
        <el-input :prefix-icon="Lock" type="password" placeholder="请输入再次密码" v-model="registerData.rePassword"></el-input>
    </el-form-item>
```
- `v-if="isRegister"`：当 `isRegister` 为 `true` 时显示此表单。
- `:model="registerData"`：将表单数据与 `registerData` 对象绑定。
- `:rules="rules"`：绑定上面定义好的校验规则。
- 每个 `el-form-item` 的 `prop` 属性与 `rules` 中的字段名和 `registerData` 中的属性名相对应，这是 Element Plus 表单校验的关键。
- 输入框通过 `v-model` 双向绑定数据，通过 `:prefix-icon` 添加图标，`type="password"` 将输入内容显示为星号。

- 点击“返回”链接时，会做两件事：将 `isRegister` 设为 `false` 切换到登录表单，并调用 `clearRegisterData` 清空注册表单数据。

## **10. 登录表单模板**
```html
<!-- 登录表单 -->
<el-form ref="form" size="large" autocomplete="off" v-else :model="loginData" :rules="rules">
    <el-form-item>
        <h1>登录</h1>
    </el-form-item>
    <el-form-item prop="username">
        <el-input :prefix-icon="User" placeholder="请输入用户名" v-model="loginData.username"></el-input>
    </el-form-item>
    <el-form-item prop="password">
        <el-input name="password" :prefix-icon="Lock" type="password" placeholder="请输入密码" v-model="loginData.password"></el-input>
    </el-form-item>
```
- 使用 `v-else`，当 `isRegister` 为 `false` 时显示登录表单。
- 结构和注册表单类似，但数据模型绑定的是 `loginData`，且没有“确认密码”字段。
- 同样复用了 `rules` 校验规则，但由于 `loginData` 中没有 `rePassword` 属性，所以 `rePassword` 的校验规则不会生效。
