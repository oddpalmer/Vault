# 项目流程说明

 ## 这个项目是什么结构

  这是一个典型的前后端分离项目。

  - 前端：Vue 3 + Vite + TypeScript
  - 后端：Spring Boot + MyBatis
  - 数据层：MySQL
  - 缓存/令牌：Redis
  - 登录认证：JWT
  - UI：Ant Design Vue
  - 状态管理：Pinia + 持久化插件

  ---

  ## 前端技术栈

  ### 核心技术
  - `Vue 3`：负责页面渲染和组件组织
  - `Vite`：前端开发服务器和构建工具
  - `TypeScript`：给前端代码加类型约束
  - `Vue Router`：页面路由跳转
  - `Pinia`：全局状态管理
  - `axios`：发送 HTTP 请求

  ### UI 和辅助工具
  - `Ant Design Vue`：表单、按钮、菜单、布局等组件
  - `pinia-plugin-persistedstate`：把 token 持久化到浏览器本地存储
  - `md-editor-v3`：文章内容编辑器
  - `less`：样式预处理

  ### 前端入口流程
  1. 浏览器打开前端页面。
  2. `main.ts` 创建 Vue 应用。
  3. 挂载 `Pinia`、`Router`、`Ant Design Vue`。
  4. `App.vue` 只负责渲染 `<RouterView />`。
  5. 路由决定显示登录页、用户页、分类页或文章页。

  ---

  ## 后端技术栈

  - `Spring Boot`：Web 服务框架
  - `Spring MVC`：Controller、拦截器、参数绑定
  - `MyBatis`：数据库访问
  - `Redis`：保存 token 的有效性
  - `JWT`：生成和解析登录令牌
  - `ThreadLocal`：保存当前请求的用户信息

  ## 单用户和多用户访问有什么区别

  ### 单用户

  - 这个用户连续发多个请求
  - 每个请求可能由不同线程处理
  - 前一个请求的局部变量不会自动给下一个请求使用

  ### 多用户

  - 多个用户同时访问，实际上是很多请求同时进来
  - Tomcat 会用线程池分配线程处理这些请求
  - 不同请求之间互不共享线程里的临时数据

  ### 关键点

  - ThreadLocal 只对“当前请求线程”有效
  - 它不是“用户级别存储”
  - 请求结束后必须 remove()，否则线程复用时可能串数据

  ———

  ## Tomcat 是怎么处理请求的

  Tomcat 不是每来一个请求就创建一个新线程，而是：

  1. 启动时准备线程池
  2. 请求来了，分配一个空闲线程
  3. 线程负责完成这次请求
  4. 请求结束，线程回到线程池继续复用

  这意味着：

  - 并发请求靠线程池处理
  - 用户越多、请求越密集，并发压力越大
  - 如果数据库慢，线程会被占住，整体响应会变慢

  ———

  ## 这个项目里的关键数据流

  ### token 流

  - 登录成功后，后端生成 JWT
  - JWT 返回给前端
  - 前端存到 Pinia
  - Pinia 持久化到本地存储
  - 之后每次请求自动放进 Authorization
  - 后端拦截器验证 token 是否还在 Redis 中
  - 验证通过后，把当前用户信息放进 ThreadLocal

  ### 用户信息流

  - token 里保存 id、username
  - 拦截器解析 token 后放入线程上下文
  - Controller / Service 通过 ThreadLocalUtil.get() 拿当前用户
  - 业务层据此查询当前用户的数据

  ## 我对这个项目的理解

  这个项目的核心不是“页面多”，而是“请求链路完整”：

  - 前端负责收集数据、做基础校验、发请求、保存 token
  - 后端负责鉴权、业务处理、数据库读写
  - Redis 用来控制 token 是否有效
  - ThreadLocal 用来让同一个请求里的代码共享当前用户信息
  - Tomcat 用线程池同时处理多个请求
