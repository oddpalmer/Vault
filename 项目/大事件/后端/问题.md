# 问题

1. application.yml没有spring标识:  要用企业版
2. Result警告：#todo
3. mapper接口有多个参数，要在参数名称前用@Param注解指定名称
4. SpringMVC@RequestBody常用于Put和Post请求,这个注解将接收的Json数据**反序列化**为Java对象，**作用于方法形参中**只能用**一次**;
5. SpringMVC@RequestParam将表单或者浏览器提交的URL键值对数据,按名字接收,**数据绑定**(类型转换)到**同名到方法形参中**;只有一个基础类型的形参可以不加，默认自动绑定。
6. Jackson@JsonIgnore把当前对象转换成Json字符串的时候，SpringMVC会忽略有该注解的字段，不输出。**作用于实体类字段上**
7. 是不是 postman post 的 JSON 数据和实体类的字段名保持一致？
	- 默认需要保持一致。若不一致，需要用 Jackson@JsonProperty等注解建立映射。
	- 只要 JSON 的字段名能匹配上 Java 对象属性的实际名或注解别名，就能接收到，且结构可以不完全一样（多字段或少字段通常都不影响）。
