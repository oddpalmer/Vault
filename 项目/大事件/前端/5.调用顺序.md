

```
📦 前端页面（发请求）
  ↓ 调用
📋 api/user.js（填快递单：地址、包裹内容）
  ↓ 调用
🚚 request.js（快递公司：贴标签、检查、派送）
  ↓ 发送
🖥️ 后端服务器（收件人）
```

---

### 代码

**1. 页面调用 api**
```javascript
import { userLoginService } from '@/api/user.js'

// 调用登录api
const result = await userLoginService({ username: 'admin', password: '123' })
```

**2. api 调用 request**
```javascript
// api/user.js
import request from '@/utils/request.js'  // ← 引入的就是 instance

export const userLoginService = (loginData) => {
    // 把数据包装一下
    const params = new URLSearchParams()
    for(let key in loginData){
        params.append(key, loginData[key])
    }
    // 调用 request 的方法，返回 Promise
    return request.post('/user/login', params)
    //     ↑ 这个 request 就是 request.js 里 export default 的 instance
}
```

**3. request.js 加工后发出去**
```javascript
// 创建 axios 实例
const instance = axios.create({ baseURL: '/api' })

// 请求拦截器：发之前加 token
instance.interceptors.request.use(
    (config) => {
        config.headers.Authorization = token  // 贴认证标签
        return config  // 交出去
    }
)

// 响应拦截器：收到后处理结果
instance.interceptors.response.use(
    result => {
        // 成功：剥掉外层，只返回 data
        return result.data  // 这个就回到了 api，api 再回到页面
    },
    err => {
        // 失败：401 就跳登录，其他报异常
    }
)

export default instance  // ← 导出被导入到 api/user.js 里
```

---

### 数据怎么流动

```
页面调用 userLoginService(data)
  ↓
api/user.js: return request.post('/user/login', params)
  ↓
request.js: instance 收到请求
  → 请求拦截器加 token
  → 发到后端 /api/user/login
  ← 后端返回 { code:0, msg:"成功", data:{token:"xxx"} }
  → 响应拦截器判断 code===0，剥离外层
  → return result.data 即 {code:0, msg:"成功", data:{token:"xxx"}}
  ↓
api/user.js: return 的 Promise 带着这个结果 resolve
  ↓
页面: const result = await...  拿到 {code:0, msg:"成功", data:{token:"xxx"}}
```

---

### 总结

**api 是传话的，request 是干活的。** api 告诉 request "用 post 访问 /user/login"，request 负责加 token、发请求、处理错误、拆包装，最后把干净数据原路返回给页面。
