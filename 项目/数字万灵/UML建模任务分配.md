# 数字万灵采购销一体化平台  UML建模任务分配（RUP简化版）

---

## 一、建模方式

本次 UML 建模采用 **RUP（统一软件开发过程）简化方式**。

RUP 强调用例驱动、迭代开发和架构中心。结合课程设计和答辩要求，本项目不画完整 4+1 视图，而是选择最能体现系统需求的核心 UML 图进行建模。

本次建议完成 **11～12 张图**：

| 图类型 | 数量 | 说明 |
|--------|------|------|
| 用例图 | 5 张 | 按五个端分别绘制 |
| 活动图 / 流程图 | 5 张 | 描述核心业务流程 |
| 类图 | 1～2 张 | 描述核心业务类和用户权限类 |
| **合计** | **11～12 张** | 适合课程设计、文档和答辩 |

---

## 二、系统划分

根据需求文档，系统分为五个端：

1. 采购企业 WEB 端
2. 采购企业移动端
3. 供应商 WEB 端
4. 供应商移动端
5. 运营管理平台 WEB 端

## 七、绘图要求

1. 每张图只画核心功能，不需要画所有按钮级操作
2. 图名、参与者名、类名保持统一
3. 每张图配 3～5 句话说明
4. 推荐使用 Draw.io、StarUML 或 ProcessOn 绘制
5. 最终导出 PNG 后放入课程设计文档或答辩 PPT
